<p class="pe-none"><a href="#" tabindex="-1" aria-disabled="true">这个链接</a> 无法点击，因为 <code>pointer-events</code> 属性是从其父级继承的。 然而, <a href="#" class="pe-auto">这个链接</a> 有一个 <code>pe-auto</code> 类，可以点击。</p>

.pe-none 类（以及它设置的 pointer-events CSS 属性）仅阻止与指针（鼠标、触控笔、触摸）的交互。默认情况下，带有 .pe-none 的链接和控件对于键盘用户仍然是可聚焦和可操作的。为了确保它们即使对于键盘用户也完全无效，您可能需要添加更多属性，例如 tabindex="-1" （以防止他们接收键盘焦点）和 aria-disabled="true" （以传达他们的事实被有效地禁用辅助技术），并且可能使用 JavaScript 来完全阻止它们的可操作性。

如果可能，更简单的解决方案是：

  • 对于表单控件，添加禁用的 HTML 属性。
  • 对于链接，删除 href 属性，使其成为非交互式锚或占位符链接。
